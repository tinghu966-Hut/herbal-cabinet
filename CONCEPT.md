# 百斗柜（Herbal Cabinet）— 概念说明书

> 受中药房百斗柜启发，设计的一套 AI 记忆管理系统

---

## 一、它是什么

中药房的百斗柜，是一面墙的木头抽屉柜子：
- 每个抽屉放一味药
- 小伙计记住每个抽屉的位置
- 掌柜根据药方抓药

**百斗柜把同一套逻辑搬到了 AI 的世界里：**

```
中药房                            →  百斗柜
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⛑️ 掌柜（知道每味药放哪）           →  🧑‍💼 掌柜（DeepSeek AI）
👦 小伙计（管一组抽屉）             →  👥 小伙计（管理同一主题的抽屉）
🗄️ 百斗柜抽屉（一味药一个抽屉）     →  📁 抽屉（一个主题的记忆）
🧑‍⚕️ 来抓药的客人                    →  🤖 来存/取记忆的 AI Agent
```

---

## 二、核心设计理念

### 原则 1：所有记忆进柜前翻译成中文

不管客人用英文、日文还是中英文混着说，掌柜先翻译成中文再存。好处：
- 存储语言统一，节省空间
- 跨语言兼容，检索时不受语言限制
- 中文作为"桥梁语言"，匹配更准

### 原则 2：分类是推导结果，不是人工标签

掌柜用 DeepSeek 理解内容后，**自动决定**放哪个抽屉。不是靠人打标签。

例：你说"八字排盘算法需要优化" → 掌柜分析 → 归入 `项目 > 东方命运 > 修复记录`

### 原则 3：路径定位 > 向量搜索

一般的 AI 记忆系统用"向量搜索"（模糊匹配），百斗柜用"走到对应抽屉翻内容"。就像去中药房抓药——你知道"当归在左边第三排第二个抽屉"，直接走过去拿，不需要满屋子闻味道。

### 原则 4：满了就分裂，不硬塞

每个抽屉最多 100 条。满了自动分裂成子抽屉，有条不紊。

## 三、系统架构

```
客人（AI Agent）："记得上次八字的bug怎么修的吗？"
          │
          ▼
┌─────────────────────────────────────┐
│           掌柜（shopkeeper）         │
│                                     │
│  ① 翻译 — 任意语言→中文             │
│  ② 分类 — 判断该放哪个抽屉           │
│  ③ 调度 — 派给对应的小伙计           │
└──────┬──────────────────┬───────────┘
       │                  │
       ▼                  ▼
┌──────────────┐  ┌──────────────┐
│ 小伙计 技术-甲 │  │ 小伙计 项目-乙 │
│              │  │              │
│ ├ AI模型      │  │ ├ 东方命运    │
│ ├ 项目部署    │  │ │  ├ 修复记录 │
│ └ ...        │  │ │  ├ 运营    │
│              │  │ │  ├ 财务    │
│              │  │ │  └ 音频    │
│              │  │ └ 咨询       │
│              │  └ ...          │
└──────────────┘  └──────────────┘
```

**当前 22 个抽屉，4 个小伙计：**

| 小伙计 | 管的抽屉范围 |
|:------|:-----------|
| 通用-甲 | 通用（1个抽屉） |
| 群聊-甲 | BLBQ公司相关（5个抽屉） |
| 技术-甲 | AI模型、项目部署等（6个抽屉） |
| 项目-甲 | 东方命运、咨询、AI咨询等（10个抽屉） |

## 四、怎么用

### 存记忆
```
node src/cli.js store "今天讨论的内容摘要"
```
掌柜自动：翻译 → 分类 → 生成摘要 → 存入对应抽屉

### 查记忆
```
node src/cli.js recall "关于八字的修复记录"
```
掌柜自动：理解问题 → 定位到正确抽屉 → 取出内容

### 交互模式
```
node src/cli.js
```
进入对话模式，直接跟百斗柜聊天存取
